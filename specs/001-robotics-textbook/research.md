# Research: Physical AI & Humanoid Robotics Textbook

**Feature**: `001-robotics-textbook`
**Date**: 2025-12-07

## 1. Docusaurus Configuration for Performance

- **Decision**: Use standard Docusaurus build optimization options.
- **Rationale**: Docusaurus 3.x is highly optimized out of the box. `FCP < 1.5s` is achievable with standard preset if images are optimized.
- **Alternatives Considered**: Custom Webpack config (rejected: keep it simple/standard).

## 2. Sidebar Architecture

- **Decision**: Multiple sidebars (`modulesSidebar`, `weeksSidebar`, `hardwareSidebar`) defined in `sidebars.ts`.
- **Rationale**: Allows distinct navigation contexts for "Topic" vs "Schedule" views as clarified in the spec.
- **Alternatives Considered**: Single autogenerated sidebar (rejected: insufficient for dual-view requirement).

## 3. Search Plugin

- **Decision**: `@docusaurus/theme-search-local`
- **Rationale**: Offline-capable, simple setup, no external dependencies (like Algolia). Fits "textbook" use case well.
- **Alternatives Considered**: Algolia DocSearch (rejected: requires external account/crawling).

## 4. Diagramming

- **Decision**: `@docusaurus/theme-mermaid`
- **Rationale**: Native support for Mermaid diagrams in markdown. Editable, version-controllable diagrams.
- **Alternatives Considered**: Static Images (rejected: hard to edit), PlantUML (rejected: external dependency).

## 5. Image Optimization

- **Decision**: Use `imagemin` or IDE plugins manually for now; consider `ideal-image` plugin if SC-006 is at risk.
- **Rationale**: Start simple. WebP/Compressed PNGs in `/static/img` are sufficient for most cases.
